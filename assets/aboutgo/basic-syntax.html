Basic Go Syntax

Line Separator

Golang does not require the use of a special line separator like ";" in Java or C. Rather, a new line key serves as a statement terminator. This can be seen in the examples to the right.

Comments

It is not surprising that Go allows two forms of comments. Go has inline comments notated by //.

// this an inline comment

And Go has block comments denoted by /* to open and */ to close. 

/* 
 This is 
 a block
 comment.
*/

Identifiers

User defined identifers must begin with a letter or and underscore and then only contain letters, numbers and digits.

identifier = letter { letter | unicode_digit }

Variable Declaration

There are multiple ways to declare variables in go. These are all demonstrated in the Variable Declaration Example to the right. Go is statically typed but feels dynamically typed since supplying types to variable declarations are not required. The var key word is used to create a new variable. The short assignment is shorthand to declare variables but can only be used within functions. Outside a function the := construct is not available.

Variables declared but not initialized with a particular value are given their zero value.

The zero value is: 0 for numeric types, false for boolean types, and "" the empty string for strings.

package main

import "fmt"

// Used var to declare and initialize word - type not required
var word string = "Potato"

func changeWord() {
	fmt.Println("Second:", word)
	// short assignment - can only be used within functions and scope is contained to function
	word := "Spud"
	fmt.Println("Third: ", word)
}

func main() {
	fmt.Println("First:", word)
	changeWord()

	// word's value returns to "Potato" after changeWord() ends.
	fmt.Println("Fourth:", word)
}
