Compilation in Go

Go is greatly influenced by C. It emphasizes on simplicity and safety.

  • Remote Package management.
  • Built-in concurrency primitives.
  • Type embedding instead of non-virtual inheritance.
  • Toolchain which produces statically linked binaries without external dependencies.

Compile is invoked as 'go tool compile'. It compiles a single Go Package which includes the files mentioned on the command line. It writes a single object file named for the basename of the first source file with a .o suffix. The object file is then compiled with other objects into a package archive or passed directly to the linker ('go tool link').

Command Line Usage

go tool compile [flags] file...

The specified files must be Go source files and all part of the same package. Same compiler is used for all target operating systems and architectures. The GOOS and GOARCH environment variables set the desired target.

Compiler Directives

The compiler in Go accepts the directives in form of comments. Directives require no space between the comment opening and the name of the directive. Examples of Line Directives:
  • //line :line
  • //line :line:col
  • //line filename:line

Why are Go compilers fast?

  • Go compiler has a simpler optimizer than C/C++ compilers.
  • Unlike C++, Go has no templates and no inline functions.
  • Go compiler generates low-level assembly code sooner and the optimizer works on the assembly code.
  • Some C/C++ compilers generate instructions in text form, while the Go compiler generates instructions in binary form.
  • A type-safe and statically-compiled language that uses seperate compilation with dependency and type checking across module boundaries to avoid unnecssary re-reading of header files and re-compiling of other modules.
